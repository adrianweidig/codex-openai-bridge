# Code of Conduct

Dieses Projekt soll ein sachlicher und respektvoller Ort für technische Zusammenarbeit sein.

## Erwartetes Verhalten

- freundlich und professionell kommunizieren
- unterschiedliche Erfahrungsstände respektieren
- Kritik auf Code, Dokumentation und Verhalten beziehen
- reproduzierbare Informationen liefern, wenn ein Problem gemeldet wird
- Sicherheits- und Datenschutzgrenzen beachten

## Nicht akzeptiertes Verhalten

- persönliche Angriffe, Beleidigungen oder Herabsetzungen
- Belästigung oder diskriminierende Sprache
- Veröffentlichung von privaten Daten, Secrets, Tokens oder Zugangsdaten
- absichtliches Stören von Issues, Pull Requests oder Diskussionen

## Durchsetzung

Maintainer können Beiträge bearbeiten, schließen oder entfernen, wenn sie gegen diese Regeln verstoßen. Wiederholte oder schwere Verstöße können zum Ausschluss von der Projektkommunikation führen.

Wenn ein Problem vertraulich gemeldet werden muss, nutze nach Möglichkeit GitHub Private Vulnerability Reporting oder Security Advisories. Sicherheitsdetails gehören nicht in öffentliche Issues.
